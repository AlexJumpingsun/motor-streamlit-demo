# 外网演示部署说明

这个仓库现在有两个入口：

- `app.py`
  继续保留为你的本地版，适合串口直连 STM32 / 本机调试。
- `app_public_demo.py`
  新增的外网演示版，适合部署到公网。它优先读取华为云 IoTDA；如果云端 secrets 没配好，会自动切到 Demo replay，页面仍然可以正常展示。

## 1. 本地先试运行外网演示版

安装依赖：

```bash
pip install -r requirements.txt
```

本地运行：

```bash
streamlit run app_public_demo.py
```

如果你想让它读取真实华为云数据，在项目根目录新建：

```text
.streamlit/secrets.toml
```

内容可以照着 `.streamlit/secrets.toml.example` 填。

## 2. 推荐部署方式

最省事的是部署到 Streamlit Community Cloud：

1. 把项目上传到 GitHub。
2. 登录 Streamlit Community Cloud。
3. New app。
4. Repository 选你的仓库。
5. Main file path 填 `app_public_demo.py`。
6. 在 App settings 里把华为云参数填到 Secrets。
7. Deploy。

部署成功后会得到一个公网链接，别人直接打开就能看。

## 3. Secrets 需要填什么

至少这几个：

```toml
HUAWEI_IOTDA_ENDPOINT = "your-iotda-endpoint"
HUAWEI_PROJECT_ID = "your-project-id"
HUAWEI_DEVICE_ID = "your-device-id"
HUAWEI_IAM_TOKEN = "your-iam-token"
HUAWEI_SERVICE_ID = "Motor"
```

说明：

- `HUAWEI_IAM_TOKEN` 如果是临时 token，过期后需要更新。
- 如果 secrets 没填，页面会自动显示演示数据，不会直接报废。

## 4. 现在这个外网演示版和原版的区别

- `app.py` 还能读本地串口 `COM` 口。
- `app_public_demo.py` 不再依赖本地串口，所以适合放到云上。
- 外网版保留了原来那套看板风格和实时曲线。
- 外网版增加了 `Demo replay`，用于没有真实云数据时做展示。

## 5. 你后面如果要做正式网站

当前这版已经够做演示站。

如果后面你想做得更正式，可以继续往下走：

- 绑定你自己的域名
- 把 IoTDA token 改成更稳的后端鉴权方案
- 把页面拆成首页 / 实时监测 / 历史记录几个页面
- 增加登录和权限控制

## 6. 当前建议

先直接部署 `app_public_demo.py`。

这样你原来的 `app.py` 完整保留，外网展示和本地调试两条线就分开了，不会互相影响。
